# 问题分解启发式指南

本指南为建模手提供从自然语言题目到数学结构的系统化分解方法。

---

## 一、变量识别协议

拿到题目后，第一步是从文本中提取三类变量：

### 1.1 决策变量（我们能控制的量）

自然语言线索：
- "确定...的最优方案" -> 方案选择就是决策变量
- "安排/分配/调度" -> 分配量是决策变量
- "选择/决定" -> 选择对象是决策变量

示例：
- "确定每个工厂的产量" -> $x_i$ = 工厂 $i$ 的产量
- "选择哪些项目投资" -> $x_i \in \{0,1\}$，1 表示投资项目 $i$

### 1.2 参数（题目给定的常数）

自然语言线索：
- 题目表格中的数据
- "已知..."、"给定..."
- 成本系数、距离矩阵、容量上限

### 1.3 状态变量（由决策变量推导的中间量）

自然语言线索：
- "...导致的..."、"...产生的..."
- 库存量、等待时间、满意度得分

### 1.4 提取工作表

| 变量名 | 符号 | 类型 | 来源 | 取值范围 |
|-------|------|------|------|---------|
| 工厂i产量 | $x_i$ | 决策 | 题意 | $[0, C_i]$ |
| 单位成本 | $c_i$ | 参数 | 附件1 | 已知 |
| 总成本 | $Z$ | 状态 | 推导 | $\geq 0$ |

## 二、约束提取清单

### 2.1 物理/资源约束

| 自然语言模式 | 数学表达 | 示例 |
|-------------|---------|------|
| "不超过/至多" | $\leq b$ | 仓库容量不超过1000吨 -> $\sum x_i \leq 1000$ |
| "不少于/至少" | $\geq b$ | 每天至少供应200件 -> $x_i \geq 200$ |
| "恰好/正好" | $= b$ | 所有需求必须满足 -> $\sum x_{ij} = d_j$ |
| "总量为" | $\sum = b$ | 预算总额为100万 -> $\sum c_i x_i = 100$ |

### 2.2 逻辑约束

| 自然语言模式 | 数学表达 |
|-------------|---------|
| "如果选A则必须选B" | $x_A \leq x_B$ |
| "AB互斥，只能选一个" | $x_A + x_B \leq 1$ |
| "至少选k个" | $\sum x_i \geq k$ |
| "A在B之前完成" | $t_A + d_A \leq t_B$ |

### 2.3 隐含约束（容易遗漏）

- 非负性：$x_i \geq 0$
- 整数性：人数、件数必须为整数
- 01性：选/不选必须为0-1变量
- 物理合理性：距离非负、概率在[0,1]之间

## 三、目标函数构建

### 3.1 判断目标类型

| 自然语言 | 目标类型 |
|---------|---------|
| "最小化成本/时间/距离" | 单目标最小化 |
| "最大化利润/效率/满意度" | 单目标最大化 |
| "在控制成本的同时提高质量" | 多目标 |
| "评价/排名/选择最优方案" | 评价问题（无显式目标函数）|
| "预测未来趋势" | 预测问题（无显式目标函数）|

### 3.2 多目标处理策略

| 策略 | 适用场景 | 方法 |
|------|---------|------|
| 加权求和 | 目标可量化、权重可确定 | $\min w_1 f_1 + w_2 f_2$ |
| 约束转化 | 某个目标有明确阈值 | $\min f_1 \text{ s.t. } f_2 \leq \epsilon$ |
| Pareto前沿 | 需要展示所有权衡方案 | NSGA-II, MOEA/D |
| 分层序贯 | 目标有明确优先级 | 先优化 $f_1$，在最优解集内优化 $f_2$ |

### 3.3 目标函数构建模板

```
min/max  Z = [系数] * [决策变量] 的某种组合

具体形式：
- 线性：Z = c^T x
- 二次：Z = x^T Q x + c^T x
- 非线性：Z = f(x)（需要说明函数形式）
```

## 四、建模范式决策树

> **决策树是引导，不是束缚**。本节的决策树只列出常见建模范式，如果题目的结构与树中所有分支都不契合，或者你从题目的物理 / 经济 / 社会背景中看出一个更合适的自创模型，**允许脱离决策树**。数学建模评奖的核心是 **数学推导的严密性 + 模型的原创性 + 问题抽象的深度**，决策树只是帮你避免遗漏常见方法。自创方法的合法性见 `assets/算法选择决策指南.md` §6。

```
题目特征分析
│
├── 数据是否含时间维度？
│   ├── 是 -> 数据量大吗(>100条)？
│   │   ├── 是 -> ARIMA / Prophet / LSTM / XGBoost
│   │   └── 否 -> 灰色预测 / 指数平滑 / 插值拟合
│   └── 否 -> 继续判断
│
├── 是否有明确的优化目标（min/max某个量）？
│   ├── 是 -> 变量是否可连续取值？
│   │   ├── 是 -> 是否凸问题？
│   │   │   ├── 是 -> LP / QP / 凸优化
│   │   │   └── 否 -> GA / PSO / SA / 模拟退火
│   │   └── 否（离散/整数）-> IP / 动态规划 / 图论
│   └── 否 -> 继续判断
│
├── 是否需要对方案/对象进行排序/评价？
│   ├── 是 -> 指标数量多吗(>5)？
│   │   ├── 是 -> 组合赋权(AHP+熵权) + TOPSIS
│   │   └── 否 -> AHP / 灰色关联 / 简单加权
│   └── 否 -> 继续判断
│
├── 是否需要发现数据中的分组/模式？
│   ├── 是 -> 是否知道分几组？
│   │   ├── 是 -> K-Means / GMM
│   │   └── 否 -> DBSCAN / 层次聚类
│   └── 否 -> 继续判断
│
├── 是否需要模拟复杂系统的动态行为？
│   ├── 是 -> 蒙特卡洛 / 元胞自动机 / 微分方程 / 排队论
│   └── 否 -> 综合分析题，组合使用多种方法
```

## 五、多问题关联分析

### 5.1 常见的问题依赖模式

竞赛题目中 4-5 个子问题通常呈现以下模式之一：

**模式A：渐进式深化**
```
Q1: 建立基线模型（简单场景）
Q2: 增加约束或扩展规模（复杂场景）
Q3: 改变条件或反向分析（灵敏度/鲁棒性）
Q4: 模型推广或策略建议（开放性）
```

**模式B：平行式多角度**
```
Q1: 从角度A分析（如经济性）
Q2: 从角度B分析（如安全性）
Q3: 综合A和B
Q4: 给出建议/预测
```

**模式C：数据驱动链**
```
Q1: 数据预处理 + 特征分析
Q2: 建模 + 预测/分类
Q3: 结果分析 + 模型检验
Q4: 策略优化 + 建议
```

### 5.2 如何识别依赖关系

1. Q(n+1) 是否使用了 Q(n) 的模型或结果？如果是，标记为"依赖"
2. Q(n+1) 是否在 Q(n) 的基础上增加了约束？如果是，标记为"扩展"
3. 两个问题是否完全独立可以并行解决？如果是，标记为"独立"

### 5.3 时间分配参考（3天竞赛）

| 阶段 | 时间 | 任务 |
|------|------|------|
| Day 1 上午 | 4h | 通读题目、问题分解、方法选择 |
| Day 1 下午 | 4h | Q1 建模 + 编码 |
| Day 1 晚上 | 3h | Q1 完成 + Q2 开始 |
| Day 2 全天 | 10h | Q2-Q3 建模 + 编码 + 结果 |
| Day 2 晚上 | 3h | Q4 + 开始写论文 |
| Day 3 上午 | 4h | 论文主体撰写 |
| Day 3 下午 | 4h | 论文完善 + 图表 + 检查 |
| Day 3 晚上 | 2h | 最终检查 + 提交准备 |

## 六、灵敏度分析协议

建完模型后，必须对以下方面进行灵敏度分析：

1. **关键参数扰动**：对每个关键参数变化 +-10%, +-20%，观察目标函数值变化
2. **约束松弛/收紧**：放松或收紧某个约束，观察最优解变化
3. **数据噪声**：在输入数据上加入随机噪声，观察结果稳定性
4. **初始条件**：改变算法初始解，观察最终解是否一致

将灵敏度分析结果用表格和图形展示，写入论文相应问题的"结果与分析"小节。
