# 数学建模技能 - Math Modeling

本技能为数学建模竞赛提供五阶段工作流：**前置准备 → 建模分析 → 代码实现 → 论文撰写 → 编译自查**，最终交付 LaTeX 论文项目及编译后的 PDF。

---

## 关键原则（Critical Rules）

所有阶段必须遵守：

1. **先读比赛规则文档，再读题目，再做任何其他事** — 页数限制、摘要长度、是否有目录、字号、附录政策等全部由规则文档决定。规则文档由用户提供，若未提供必须主动索要。
2. **读题后立即把原题导出为 markdown 文件**（如 `C题.pdf` → `C题.md`），后续所有阶段都基于此 md 文件对齐题意。
3. **文献检索是必做任务**，不是可选项。建模手与论文手都必须独立使用 `paper_search` 工具完成检索，并将命中文献写入产出文档。
4. **每一问必须"先对比多方法，再建模"**：
   - 建模阶段自主提出 ≥ 3 种候选方法 → 用对比表筛选出 ≥ 2 种进入论文
   - 论文中每一问的第一小节就是 **方法对比与选择**，再展开建模 / 求解 / 结果分析
5. **论文交付形态是 LaTeX 项目**，编译为单个 PDF，不交付 Word / Markdown / HTML。
6. **排版失败即扣分**：禁止出现"一页一张图 + 下方整片空白"、摘要溢出到第二页、章节标题孤悬页底等问题。排版最佳实践见 `references/latex-guide.md`。
7. **按规则文档控制篇幅**：正文页数、摘要长度、附录政策以比赛规则为准；没有规则文档时默认采用国赛惯例（正文 ≤ 25 页 + 摘要独占一页 + 参考文献和附录不计页数），但必须在产出中声明"采用默认值，待规则文档确认"。
8. **建模创新 > 算法复杂度**：数模竞赛的评分核心是**数学推导的严密性 + 模型的原创性 + 问题抽象的深度**，而不是"用了多新的算法"。候选方法不必非得从 `assets/` 算法目录里挑，**允许并鼓励**：
   - **自己推导新模型**（自建微分方程、自建效用函数、自建概率分布、自建评价指标等）
   - **组合 / 改造基础方法**（如 灰色预测 + ARIMA 残差修正、改造 TOPSIS 的距离度量、PSO 惯性权重改进）
   - **跨领域嫁接**（如把物理中的"场"概念用到人群流动建模）
   - **使用从未有人用过的方法**，只要 ① 数学推导成立、② 假设明确、③ 结果可复现
   
   **唯一不允许的是造假**（数据造假、结果编造、引用造假）。**不要为了"显得高级"而硬套 LSTM / XGBoost / Transformer**，用经典方法 + 严密推导反而容易得高分。

## 工作流总览

```
阶段零 前置准备
  ├── 0.1 获取并解析比赛规则文档（由用户提供）
  ├── 0.2 读题 → 导出 markdown → 检查附件 / 附录数据
  └── 0.3 根据题型判断要读的算法文档
        ↓
阶段一 建模分析     ── 产出：题目分析报告.md、术语表格.md
        ↓
阶段二 代码实现     ── 产出：q1.py ~ qN.py、output/（csv + log + figures/）
        ↓
阶段三 论文撰写     ── 产出：paper/ LaTeX 项目
        ↓
阶段四 编译自查     ── 产出：paper/main.pdf + 自查清单通过
```

## 阶段零：前置准备（Preparation，必须最先执行）

### 0.1 比赛规则文档（Rules Document）

**第一件事**：询问用户是否提供了比赛规则文档（通常是 PDF 或竞赛官方通知）。

如果 **有**：使用 `tools/pdf` 读取，导出为 `比赛规则.md`，并从中提取以下关键信息，写入项目根目录的 `规则摘要.md`：

| 必读项 | 示例（国赛） | 影响 |
|-------|-------------|------|
| 论文页数上限 | 正文 ≤ 25 页 | 控制正文篇幅、图表密度 |
| 摘要长度 | 一页、约 300-500 字 | 模板中的 `abstract.tex` 结构 |
| 是否要目录 | 国赛不要、美赛要 | `main.tex` 中 `\tableofcontents` 开关 |
| 字号 / 行距 / 边距 | 小四 / 1.5 倍 / 2.5cm | `main.tex` 的 `geometry` / `\linespread` 设置 |
| 附录政策 | 代码放附录、不计页数 | 附录的组织 |
| 参考文献格式 | GB/T 7714 / IEEE / APA | `.bib` 样式与 `\bibliographystyle` |
| 语言 | 中文 / 英文 | `ctexart` vs `article` |
| 文件命名与提交形式 | 如：承诺书+论文+源码压缩包 | 最终打包 |

如果 **没有**：**必须主动提醒用户上传**。未确认时沿用国赛默认值，并在产出文档顶部写明"以下约束基于国赛默认规则，待比赛规则文档确认后可能调整"。

**用户自定义论文模板（第二来源）**：除了比赛规则文档，用户还可能提供自己的论文模板或格式要求（如工作目录下的 `format.md` / `格式要求.md` / 学校 LaTeX 模板）。读取流程和比赛规则文档相同，导出为 `用户模板摘要.md`。

**格式优先级（从高到低，不得违反）**：

1. **比赛规则文档**（主办方规定，客观约束）
2. **用户自定义模板**（用户偏好，如学校模板）
3. **本技能默认 LaTeX 模板**（`references/latex-template/`，fallback）

例如：比赛要求正文 ≤ 25 页，但用户模板默认 30 页时，必须按 25 页执行。

### 0.2 读题并导出 markdown

使用 `tools/pdf` 读取题目 PDF，**立即**导出为 markdown 文件（如 `C题.pdf` → `C题.md`），保留原题中的所有表格、公式、图片引用、附件清单。后续所有阶段以此 md 为真源。

同步检查是否有附件目录（`附录/`、`附件/`、`appendix/`、`data/` 等），若有，用 `tools/xlsx` 粗读每个 Excel 文件，记录行列数、列名、数据类型，写入 `附件清单.md`。

### 0.3 按题型定位必读文档

读题后根据题目类型，**针对性选读** `assets/` 下的算法文档，而不是一股脑全读：

| 题目类型关键词 | 定点阅读 |
|---------------|---------|
| 优化 / 调度 / 分配 / 最大化最小化 | `assets/01-优化算法说明.md` + `assets/算法选择决策指南.md` §1 |
| 预测 / 时间序列 / 未来趋势 | `assets/02-预测类算法说明.md` + 决策指南 §2 |
| 评价 / 排序 / 方案选优 | `assets/03-评价类算法说明.md` + 决策指南 §3 |
| 路径 / 网络 / 运输 | `assets/04-图论与网络分析算法说明.md` |
| 聚类 / 分类 / 降维 / 检验 | `assets/05-统计分析与数据处理算法说明.md` |
| 仿真 / 博弈 / 微分方程 / 马尔科夫 | `assets/06-综合类算法说明.md` |
| 机器学习 / 集成学习 / 异常检测 | `assets/07-机器学习算法说明.md` |

**无论题型是什么**，都必须通读：
- `references/problem-decomposition.md` — 变量 / 约束 / 目标的系统提取协议
- `assets/算法选择决策指南.md` — 反模式清单 + head-to-head 对比

## 阶段一：建模分析（Modeling Analysis）

**先读**：`references/roles/建模手说明.md`

### 1.1 问题分解

按 `references/problem-decomposition.md` 的协议系统化执行：
- 填写变量工作表（决策 / 参数 / 状态）
- 提取三类约束（物理 / 逻辑 / 隐含）
- 判定目标类型（单目标 / 多目标 / 评价 / 预测）
- 画出各问依赖图（依赖 / 扩展 / 独立）

### 1.2 多方法候选（核心流程）

**每一问必须独立执行以下流程**：

```
Step 1  头脑风暴 ≥ 3 种候选方法
        ├── 经典方法：参考 assets/ 算法目录（规划、预测、评价、图论、统计、综合、机器学习）
        ├── 组合方法：把 2+ 个基础方法拼起来（如 灰色预测 + ARIMA 残差修正）
        ├── 改造方法：改进经典算法的某个步骤（如 改造 TOPSIS 的距离度量）
        └── 自创方法：从题目物理/经济/社会含义出发，自己推导新模型
                     （例如自建微分方程、自建效用函数、自建评价指标）
Step 2  用对比表筛选出 ≥ 2 种进入论文
        └── 维度：适用性 / 精度 / 效率 / 可解释性 / 数据要求 / 创新性
Step 3  从 ≥ 2 种中选定 1 种作为"主方法"用于实际求解
        └── 另外 1+ 种作为论文中的"对照方法"或备选
Step 4  在分析文档中记录：为何淘汰其余候选、为何选定主方法
```

**关键点**：
- **候选方法池不限于 `assets/` 算法目录**。`assets/` 是参考百科，不是必选项清单。鼓励自创、组合、改造（见关键原则 #8）。
- 禁止所有问都用同一种方法（反模式）
- **不要为了"看起来高级"硬凑 LSTM / XGBoost / Transformer**。数据量 < 500 时这类方法几乎必定过拟合，反而扣分
- 论文中既要写"主方法"的完整建模求解，也要在 **方法对比与选择** 小节中给出被淘汰 / 对照方法的简要说明，形成对比论证
- "对照方法"不一定要实际跑通，但必须给出对比表和足够的原理说明（让评委看懂为原则）
- **自创方法的合法性门槛**：① 数学推导在论文中能走通 ② 所有假设明确列出 ③ 代码实现后结果可复现 ④ 给出被淘汰候选的对比论证。满足这四条，没有任何"被允许的方法清单"限制

### 1.3 文献检索（必做）

使用 `tools/paper_search` 为每一问的主方法搜索 ≥ 3 篇文献：

```bash
python tools/paper_search/scripts/openalex_scholar.py --query "grey prediction GM(1,1)" --email "example@example.com"
```

**邮箱说明**：OpenAlex API 只要求 `mailto` 字段存在，**不验证邮箱真实性**。脚本已默认 `example@example.com`，无需问用户，直接跑。用户主动提供真实邮箱时可覆盖。

把命中文献写入 `题目分析报告.md` 的"参考文献信息"章节：论文名、作者、年份、期刊、DOI。这些文献后续进入 `references.bib`。

### 1.4 术语表

建立 `术语表格.md`（中文名 / 英文名 / 缩写 / 数学符号），后续所有阶段必须遵守此术语表，避免同一概念多种叫法。

| 文件 | 内容 |
|-----|------|
| `X题.md` | 阶段零导出的原题 md |

## 阶段二：代码实现（Coding Implementation）

**先读**：`references/roles/编程手说明.md`

### 2.1 语言确认

编写任何代码前先确认编程语言（Python / MATLAB），用户未指定时必须主动询问。

### 2.2 标准目录结构

```
project/
├── data/                 只读输入数据（题目附件）
├── q1.py                 问题一：独立可跑
├── q2.py                 问题二：独立可跑
├── ...                   qN.py
├── output/               代码的所有输出都在这里
│   ├── q1_*.csv          数值结果（论文表格从这里摘抄）
│   ├── q1_log.txt        运行日志（收敛过程、约束检查、诊断）
│   ├── q2_*.csv
│   ├── q2_log.txt
│   └── figures/          可视化图（PNG，300 dpi）
│       ├── q1_fig1.png
│       ├── q1_fig2.png
│       └── ...
└── README.md             复现说明（环境、运行顺序、依赖）
```

### 2.3 代码职责（纯粹原则）

**代码只做三件事：计算 → 出图 → 输出数值和日志到 output/。**

严禁代码越界承担论文手 / 建模手的工作：

- ❌ 不得硬编码任何**结论性判断**（如 `optimal_method = "A"`、`# 方法A更优`）
- ❌ 不得硬编码**预设结果**或从论文反推数字让代码去凑
- ❌ 不得自动生成论文片段、summary.md、对比汇报之类的文档
- ❌ 不得在 `print` / 注释 / 文件里写结论性话语（如"本方法优于 XX"）
- ✅ 只 `print` 数值 + 标签（如 `print(f"method_A obj={val:.3f}")`）
- ✅ 让结果在 `output/*.csv` 里自然出现，由建模手 / 论文手解读后再决定怎么写论文

### 2.4 可复现（必做）

- 所有随机数固定 seed：`np.random.seed(42)` / `random.seed(42)` / `torch.manual_seed(42)` 等
- 超参集中在文件顶部 `CONFIG = {...}` 块，不散落在代码中间
- 脚本必须能从空的 `output/` 目录一键跑通（`python q1.py` 就够了）
- 用相对路径 `Path(__file__).parent / "data" / "xxx.csv"`，不写绝对路径

### 2.5 其他关键约束

- **严格按建模手方案实现**：禁止因为报错就换算法、简化约束、创建 "简化版" 文件
- **充分可视化（图多胜图少）**：数模竞赛评分倾向视觉丰富。典型每问 **4-8 张图为常规范围**，简单题**不少于 3 张**，复杂题 **10+ 张也合理**，**上不封顶**。图表类型由 AI 根据题目特征自主决定（不强制 3D / 对比图等特定类型）。**唯一禁止**：为凑数的无信息价值装饰图。
- **结果验证**：量级检查 / 边界测试 / 基线对比 / 约束满足 / 预测模型做交叉验证
- **Excel 表格**：题目输入表只读，结果模板保持结构不变（只填数据）

### 阶段二产出

`q1.py ~ qN.py` + `output/q*.csv` + `output/q*_log.txt` + `output/figures/q*_fig*.png` + `README.md`

## 阶段三：论文撰写（Paper Writing）

**先读**：
- `references/roles/论文手说明.md`
- `references/latex-guide.md`（**必读**：含排版最佳实践、避免空白页的技术细节）

### 3.1 论文结构

```
[标题页]
[摘要页]                 ← 独占一页，页面内容只有摘要 + 关键词
[目录]                   ← 按比赛规则决定是否保留
§1 问题重述
§2 问题分析              ← 整体思路 + 各问依赖图
§3 模型假设
§4 符号说明
§5 问题一的模型建立与求解
  5.1 方法对比与选择     ← ≥ 2 种方法对比表 + 各自原理 + 选定理由
  5.2 模型建立           ← 决策变量 / 目标函数 / 约束；自创或困难方法必须完整推导
  5.3 求解过程
  5.4 结果与分析         ← 引用 output/ 里的数值做表格、引用 figures/ 里的图 + 分析 + 灵敏度
§6 问题二的模型建立与求解 （同 §5 结构）
...
§N 模型评价与推广
[参考文献]
[附录]                   ← 完整代码（lstlisting）+ 运行数据（longtable）
```

### 3.2 摘要独占一页（硬性要求）

摘要页必须是**独立的一整页**，不允许混入标题、目录、正文。技术实现见 `references/latex-guide.md` §"摘要独占一页"。

摘要内容结构：**问题 → 方法 → 每问具体数值结果 → 结论 → 关键词（3-5 个）**。

### 3.3 每问结构（严格遵守）

每问第一小节就是 **方法对比与选择**，即使建模阶段已经对比过，论文中也必须重新展开对比，原因：
- 评委只读论文，不读你的内部分析文档
- 方法对比本身就是论文的核心论证，体现思维过程

对比表模板见 `references/latex-template/question1.tex`。

### 3.4 图表规范

- 图片格式用 **PNG**（300 dpi），从 `output/figures/` 直接引入
- 每张图下方必须有**足够的分析**（描述具体趋势 / 拐点 / 峰值 / 对比差异）。不强制字数，但**禁止只写 "如图所示结果良好" 这种空话**——核心图多分析、辅助图可简短
- 表格使用 `booktabs` 三线表，禁止竖线
- 图表编号在正文中必须被 `\ref{}` 引用，顺序与出现顺序一致

### 3.5 数学推导（论文核心含金量）

数模评奖的核心不是"用了多高级的算法"，而是**数学推导的严密性 + 模型的原创性 + 问题抽象的深度**。所以：

- **经典方法**：关键步骤要写出来，不能只说"用 XX 方法求解"就跳过推导
- **自创 / 组合 / 改造方法**：必须完整写出数学形式、推导过程、所依赖的假设、与经典方法的差异
- **困难点**：涉及非平凡代数变换、近似、渐进分析的地方不能跳，读者/评委要能跟着走通
- 正文出现的每个符号都要有定义、每个假设都要被真实用到、每个结论都要可推导或可验证

### 3.6 去 AI 味

全文禁止出现的词汇 / 句式（详见 `references/roles/论文手说明.md`）：
- 值得注意的是、综上所述、本文创新性地、深入探讨、进一步地、不仅...而且、总而言之、深刻地、持久的、充分展示、奠定基础
- 连续 3 段以上相同句式开头
- 表情符号、装饰性粗体、破折号的炫技使用

### 阶段三产出

`paper/` 项目目录（基于 `references/latex-template/` 复制），编译产物 `paper/main.pdf`。

## 阶段四：编译与交稿自查（Compilation & Self-Check）

### 4.1 编译

```bash
cd paper/
latexmk -xelatex main.tex
```

报错时读 `main.log`，修 `.tex` 后重编译，循环直到零错误产出 `main.pdf`。

### 4.3 排版失败的修复策略

当出现"一页一图 + 大片空白"时，优先级从高到低尝试：

1. 把图片 `[H]` 改为 `[htbp]`，允许 LaTeX 自主浮动
2. 给图片上方增加文字描述段落，避免整页只有图
3. 将相邻两图合并为 `subcaption` 左右排布
4. 适当调整图片 `width`（0.6–0.85\textwidth）
5. 必要时用 `\FloatBarrier`（需 `placeins` 宏包）强制浮动体在本节内落位

详细方案见 `references/latex-guide.md` §"浮动体与空白页避坑"。
