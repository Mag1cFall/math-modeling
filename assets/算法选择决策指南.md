# 算法选择决策指南

本指南提供 head-to-head 算法对比、反模式清单和经过验证的组合方案，帮助建模手在候选算法之间做出正确决策。

本文件与 `assets/01-07` 算法说明文档配合使用：说明文档提供**算法原理参考**，本文件提供**选择决策支持**。

---

## 一、优化算法对比

### 1.1 精确算法 vs 启发式算法

| 维度 | 精确算法 (LP/IP/QP) | 启发式 (GA/PSO/SA) |
|------|---------------------|-------------------|
| 最优性保证 | 全局最优 | 近似最优 |
| 问题规模 | 中小规模（变量 < 1000） | 大规模均可 |
| 约束处理 | 原生支持 | 需罚函数/修复策略 |
| 编程难度 | 低（调库） | 中（需调参） |
| 可解释性 | 高 | 低 |
| **选择原则** | **能用精确算法就不用启发式** | **精确算法解不动时再用** |

### 1.2 启发式算法之间

| 维度 | GA（遗传算法） | PSO（粒子群） | SA（模拟退火） |
|------|---------------|--------------|--------------|
| 搜索能力 | 强（交叉+变异） | 中（群体协作） | 中（单点搜索） |
| 收敛速度 | 中 | 快 | 慢 |
| 参数调节 | 交叉率/变异率/种群 | 惯性权重/学习因子 | 温度/冷却率 |
| 离散问题 | 天然适合 | 需离散化处理 | 天然适合 |
| 连续问题 | 需编码 | 天然适合 | 天然适合 |
| 多目标 | NSGA-II 成熟 | MOPSO 可用 | 不擅长 |
| **最佳场景** | 组合优化、TSP | 连续参数寻优 | 调度问题 |

### 1.3 规划算法选择

```
变量类型？
├── 全部连续 -> 目标/约束是否线性？
│   ├── 是 -> LP (scipy.optimize.linprog / pulp)
│   └── 否 -> 是否凸？
│       ├── 是 -> QP / 凸优化 (cvxpy)
│       └── 否 -> NLP (scipy.optimize.minimize) 或启发式
├── 含整数 -> 规模多大？
│   ├── 小 (<100个变量) -> IP (pulp / ortools)
│   └── 大 -> 启发式 + 整数修复
└── 全部0-1 -> 指派/背包问题？
    ├── 是 -> 专用算法（匈牙利/动态规划）
    └── 否 -> 0-1 IP (pulp)
```

## 二、预测算法对比

### 2.1 按数据量选择

| 数据量 | 推荐方法 | 不推荐 |
|-------|---------|--------|
| < 10 条 | 灰色预测 GM(1,1) | 一切ML方法 |
| 10-50 条 | 插值拟合 / 简单回归 | 神经网络 |
| 50-500 条 | ARIMA / 指数平滑 / 多元回归 | 深度学习 |
| 500-5000 条 | XGBoost / LightGBM / Prophet | 简单灰色预测 |
| > 5000 条 | LSTM / Transformer / XGBoost | 灰色预测 |

### 2.2 时间序列方法对比

| 维度 | ARIMA | Prophet | LSTM |
|------|-------|---------|------|
| 数据量要求 | 50+ | 100+ | 500+ |
| 季节性处理 | 手动（SARIMA） | 自动 | 需特征工程 |
| 多变量 | 不支持（单变量） | 不支持 | 支持 |
| 非线性 | 不支持 | 部分支持 | 支持 |
| 可解释性 | 高 | 高 | 低 |
| 编程难度 | 中（定阶） | 低（自动） | 高 |
| **竞赛推荐度** | 首选 | 备选 | 数据充足时选 |

### 2.3 回归 vs 机器学习

| 维度 | 线性回归 | 随机森林 | XGBoost |
|------|---------|---------|---------|
| 数据量 | 任意 | 100+ | 200+ |
| 特征数 | 少（< 10） | 多均可 | 多均可 |
| 可解释性 | 高（系数有意义） | 中（特征重要度） | 中 |
| 过拟合风险 | 低（但欠拟合风险高） | 低 | 中（需调参） |
| **竞赛建议** | 优先尝试 | 线性效果差时选 | 线性效果差时选 |

## 三、评价算法对比

### 3.1 赋权方法

| 维度 | AHP | 熵权法 | 组合赋权 |
|------|-----|--------|---------|
| 类型 | 主观 | 客观 | 主观+客观 |
| 数据要求 | 不需要数据 | 需要指标数据 | 两者都需要 |
| 专家依赖 | 高 | 无 | 中 |
| 适合场景 | 指标少、有专家经验 | 指标多、有数据 | 两者兼顾时 |
| **竞赛推荐** | 单独使用偏主观 | 单独使用偏客观 | **AHP+熵权组合最优** |

### 3.2 综合评价方法

| 维度 | TOPSIS | 灰色关联 | DEA |
|------|--------|---------|-----|
| 适用结构 | 多方案多指标 | 多方案多指标 | 多投入多产出 |
| 基准需求 | 需要理想解 | 需要参考序列 | 不需要基准 |
| 指标数量 | 任意 | 任意 | 投入+产出 >= 方案数/3 |
| 结果形式 | 贴近度排名 | 关联度排名 | 效率值 |
| **最佳场景** | 通用排序 | 与标杆比较 | 效率评价（如银行/医院） |

## 四、反模式清单

以下是竞赛中常见的错误选择：

| 反模式 | 为什么错 | 正确做法 |
|--------|---------|---------|
| < 100 条数据用神经网络 | 数据不足以训练，严重过拟合 | 灰色预测 / 回归 / 插值 |
| < 5 个指标用 PCA 降维 | 维度已经够低，PCA 无意义 | 直接分析 |
| 单目标问题用 NSGA-II | 多目标算法解单目标效率低 | LP/IP 或单目标 GA |
| 有解析解的问题用启发式 | 浪费计算资源且不保证最优 | 直接解析求解 |
| AHP 单独赋权不做客观验证 | 纯主观，说服力不足 | AHP+熵权组合赋权 |
| 套用模型不检验假设 | ARIMA 要求平稳性，不检验就用 | 先做 ADF 检验等假设验证 |
| 所有问题都用同一种方法 | 显得方法单一，缺乏针对性 | 每问根据特征选择最优方法 |
| 选择评委看不懂的复杂方法 | 降低可读性和评分 | 能简单就简单 |

## 五、经过验证的组合方案

### 5.1 评价类组合

**AHP+熵权 -> TOPSIS**
- AHP 确定主观权重、熵权法确定客观权重
- 用乘法归一化或线性组合得到综合权重
- 以综合权重输入 TOPSIS 排序
- 适用于：方案选优、绩效考核

### 5.2 预测类组合

**灰色预测 + 残差修正**
- GM(1,1) 做初步预测
- 对残差序列用 ARIMA 或回归修正
- 最终预测 = GM预测 + 残差修正
- 适用于：小样本趋势预测

**ARIMA + XGBoost**
- ARIMA 捕捉线性趋势和季节性
- XGBoost 捕捉非线性残差
- 融合两者的预测结果
- 适用于：中等数据量的复杂时间序列

### 5.3 优化+评价组合

**多目标优化 + TOPSIS 决策**
- 用 NSGA-II 生成 Pareto 前沿
- 用 TOPSIS 从 Pareto 前沿中选择折中解
- 适用于：多目标决策问题

### 5.4 聚类+分类分析组合

**K-Means 聚类 + 特征分析**
- 先用 K-Means (或层次聚类) 将对象分组
- 用肘部法则确定最优 K 值
- 对每组进行统计特征分析
- 适用于：数据分析类问题

## 六、创新方法与自拼组合（重要）

本文档的对比表和组合方案只是**参考起点，不是封闭清单**。数学建模竞赛评奖的核心是 **数学推导的严密性 + 模型的原创性 + 问题抽象的深度**，所以：

- 本指南里的方法你可以不用
- 没列在这里的方法（甚至从来没人发表过的方法）你也可以用
- **只要数学推导成立、假设明确、结果可复现**，就是合法的候选方法

### 6.1 四类合法的"非现成"方法

| 类型 | 做法 | 举例 |
|-----|------|------|
| **自创模型** | 从题目的物理 / 经济 / 社会含义出发，推导一套新的数学关系 | 为题目场景自建微分方程组；自建效用函数；自建概率分布；自建评价指标 |
| **组合方法** | 把 2+ 个基础方法串联或并联 | GM(1,1) + ARIMA 残差修正；AHP × 熵权组合赋权 → TOPSIS；K-Means + DEA 分组效率评价 |
| **改造经典算法** | 在经典算法的某个关键步骤上做改进 | TOPSIS 的欧氏距离改成马氏距离；PSO 的惯性权重改成自适应；AHP 判断矩阵一致性自动修复 |
| **跨领域嫁接** | 把其他学科的成熟概念搬到本题的建模场景 | 物理"场"概念用到人群流动；热力学熵用到数据分散度；排队论用到医院床位 |

### 6.2 创新方法 vs 本指南"反模式"的辨析

本指南 §4 的反模式清单（"< 100 条数据用神经网络"等）依然适用。**反模式针对的是"错误使用"现成方法**，不针对"自创方法"。具体：

- ❌ 反模式：硬套一个不合适的现成方法（数据不够还要上 LSTM）
- ✅ 合法：从题目出发自己推导一个新模型（数学推导严密即可）

两者的区别在于：**反模式是为了"看起来高级"套用错的方法**，创新方法是**为了解决问题而自己发明方法**。目的不同。

### 6.3 写入论文的合法性门槛

自创 / 组合 / 改造方法要写入论文，必须全部满足：

1. **完整的数学形式**：在 "模型建立" 小节写出决策变量、目标函数、约束条件 / 状态方程的完整数学表达
2. **清晰的推导**：关键步骤的推导过程在论文中可见（过于显然的步骤可简略，核心步骤不能跳）
3. **明确的假设**：所有依赖的假设列入 `assumptions.tex`，每条假设在后续建模 / 求解中确实被用到
4. **可复现的实现**：代码能跑，相同输入得相同输出
5. **对比论证**：在 **方法对比与选择** 小节中给出被淘汰的对照方法，说明为什么自创方法更合适

满足这五条，方法是不是"从没人用过"都不重要。

### 6.4 反例：过度追求"算法新颖"的失败模式

| 反例 | 为什么失败 | 更好的做法 |
|-----|-----------|-----------|
| 数据 50 条，硬上 Transformer | 严重过拟合，结果不可信 | 灰色预测 / 回归 / 插值 |
| 用 GAN 做数据增强再跑 SVR | 增强数据的分布和真实分布差异大，结果是错觉 | 直接交叉验证原数据 |
| 把 4 个优化算法都跑一遍比较 | 显得没思路，评委看不出你懂哪个 | 选 1-2 个用透，深入分析 |
| 套了一个听起来很新的元启发式算法（如海马优化算法） | 除了名字新，没有比 GA/PSO 更好的理由 | 用 GA/PSO 但在参数策略上创新 |
| 所有问都用 XGBoost | 缺乏针对性，像"一把锤子看什么都是钉子" | 每问根据题目特征选不同方法 |

核心原则：**方法为问题服务，不是问题为方法服务**。
